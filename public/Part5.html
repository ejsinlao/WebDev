Part 5

Conclusion

Q: Rate your experience with the frameworks that you chose.

React: 7/10
Rails: 3/10

Did they perform to your expectations?

The frameworks did not perform to our expectation because there were some issues in regards to installation and configuration

Were they difficult to install or configure?

Ruby on Rails provided some difficulty due to the bugs of the latest Rails version. Alot of issues were created because we had to make sure the Rails and Ruby versions were compatible. Some of us even had to switch computers in order to get the Rails server running.

Was it easy to create the page(s) with it?

The pages were easy to create because we knew HTML/CSS/JS. Using the React framework on top of that helped expedite alot of our work. The Rails backend was easy as well due to the simple nature of the database.

Was the learning curve steep compared with regular HTML/CSS/Javascript/PHP?

The learning curve from installation and designing is definitely steeper compared to HTML/CSS?JS. A big part was all of the different components that come with Rails and making sure all of them work seamlessly.

Would you have done things differently in retrospect?

We would have prioritized reading documentation because Rails and React were frameworks we have not used before.

Did you regret your choice of frameworks?

We do regret the choice of our frameworks simply because we did not have alot of time to learn them. We could have used frameworks we were more familiar with like Angular and Django but decided to learn new frameworks instead. This along with installation difficulties from Rails made the experience more painful that it had to be.
